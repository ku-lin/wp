### 求时间复杂度

```cpp
void f(int p){
    int i=1;
    while(i <= n){
        i = i * 2;
    }
}
```

O（log n）

时间复杂度是O(f(n))，表示随问题规模n的增大，算法执行时间的增长率和f（n）的增长率相同，称作善法的时间复杂度

观察一下这个函数，上限是n，i每次乘以2，说明这个循环需要经过的次数是log2(n)次，那么答案就是O(log2(n))

因为时间复杂度是表示增长率的，所以一切不需要的系数和其他东西都不需要，这里的以2为底就不需要写，所以直接写O(log n)

说一下为什么这样写，因为在比较算法时间时间消耗的时候，我们往往只需要比较时间复杂度就可以了，在系数不同或者加减了什么东西的情况下可以直接忽略

再给一个例子讲解一下时间复杂度的计算方式

```cpp
#include<cstdio>
int zuziyi(int i){
    if(i == 0){
        return i;
    }
    printf("%d",zuziyi(i-1))
    printf("%d",zuziyi(i-1))
    return i;
}
int main(){
    int n=114514;
    int i=0;
    for(i=0;i*i <= n;i++){
        zuziyi(i);
    }
}
```

一步步看，看这个for循环

这个循环判断条件是i\*i <= n，那么就是直接sqrt(n)次  *这个sqrt是直接对n开根号，这里我不会打。。。*

所以循环的时间复杂度是`O(sqrt(n))`

然后再看一下函数是什么

每一次递归都会将i减一，但是会创建两个函数，相当于执行了2^n次

所以函数的时间复杂度是`2^n`

总的时间复杂度直接带进去算一下就出来了，是`O(2^√n)`

### 单链表中，已知q所指结点是p所指结点的前驱结点，若在q和p之间插入一个结点s，则执行（         ）。

A. s->next=p->next; p->next=s;

答案:C:q->next=s; s->next=p;;

在单链表指针`p`所指节点后插入新节点`s`：

1. `s->next = p->next;` // 新节点 s 的后继指向 p 原来的后继节点
2. `p->next = s;` //p 的后继指向新节点 s

实际题目可能变，我不太确定

#### <font style="color:rgb(0, 0, 0);">1. 插入前的链表状态</font>

单链表的节点结构：每个节点包含「数据」和「指针（next）」，指针指向下一个节点。假设插入前链表结构如下（`p`指向中间某节点，`p->next`指向节点`q`）：

```plain
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 节点A   │    │ 节点p   │     │ 节点q   │    （后续节点）
│ data    │    │ data    │    │ data    │
│ next →  │───>│ next →  │───>│ next →  │───> ... 
└─────────┘    └─────────┘    └─────────┘
                ↑
                指针p
```

新节点`s`初始状态：`s->next`为NULL，待插入到`p`和`q`之间。

#### <font style="color:rgb(0, 0, 0);">2. 第一步：</font>`s->next = p->next;`

将`s`的`next`指针指向`p`原来的后继节点`q`，此时`s`与后续链表建立连接：

```plain
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 节点A   │    │ 节点p   │     │ 节点s   │    │ 节点q   │
│ data    │    │ data    │    │ data    │    │ data    │
│ next →  │───>│ next →  │    │ next →  │───>│ next →  │───> ... 
└─────────┘    └─────────┘    └─────────┘    └─────────┘
                ↑                ↑
                指针p            新节点s
```

此时`p`的`next`仍指向`q`，链表原有结构未被破坏。

#### <font style="color:rgb(0, 0, 0);">3. 第二步：</font>`p->next = s;`

将`p`的`next`指针指向新节点`s`，完成`p`到`s`的连接，最终插入完成：

```plain
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 节点A   │    │ 节点p   │    │ 节点s   │    │ 节点q   │
│ data    │    │ data    │    │ data    │    │ data    │
│ next →  │───>│ next →  │───>│ next →  │───>│ next →  │───> ... 
└─────────┘    └─────────┘    └─────────┘    └─────────┘
                ↑                ↑
                指针p            新节点s
```

# 栈

先讲解一下栈是什么

栈是一种**后进先出（LIFO, Last In First Out）** 的线性数据结构，只能在一端（称为「栈顶」，Top）进行插入（入栈 / Push）和删除（出栈 / Pop）操作，另一端（栈底，Bottom）固定不动。可以类比生活中的「弹夹」「叠盘子」：最后放上去的盘子，最先被拿下来。

#### <font style="color:rgb(0, 0, 0);">二、栈的核心操作</font>

1. **入栈（Push）**：将元素添加到栈顶；
2. **出栈（Pop）**：从栈顶移除元素（栈空时无法出栈）；
3. **查看栈顶（Peek）**：仅查看栈顶元素，不删除。

#### <font style="color:rgb(0, 0, 0);">三、三步例子 + 图示讲解</font>

以「依次入栈元素 A→B→C，再出栈一次」为例，分三步演示栈的变化：

#### <font style="color:rgb(0, 0, 0);">栈的基础结构（空栈）</font>

栈底固定，栈顶指针（Top）初始指向栈底（空栈状态）：

```plain
┌─────────┐  栈空间（向上延伸）
│         │
│         │
│         │
│         │  ← 栈顶（Top）= 栈底（Bottom）
└─────────┘
```

#### <font style="color:rgb(0, 0, 0);">第一步：入栈元素 A（Push A）</font>

将 A 放到栈顶，栈顶指针上移指向 A：

```plain
┌─────────┐
│         │
│         │
│         │
│    A    │  ← 栈顶（Top）（元素A：栈底=栈顶）
└─────────┘
          ↓
        栈底（Bottom）
```

此时栈内元素：\[A]（栈底→栈顶）

#### <font style="color:rgb(0, 0, 0);">第二步：入栈元素 B（Push B）</font>

将 B 放到当前栈顶（A 上方），栈顶指针继续上移指向 B：

```plain
┌─────────┐
│         │
│         │
│    B    │  ← 栈顶（Top）
│    A    │
└─────────┘
          ↓
        栈底（Bottom）
```

此时栈内元素：\[A, B]（栈底→栈顶）

#### <font style="color:rgb(0, 0, 0);">第三步：入栈元素 C（Push C）→ 再出栈一次（Pop）</font>

1. 先入栈 C：将 C 放到 B 上方，栈顶指针指向 C；\*\*\*\*

```plain
┌─────────┐
│         │
│    C    │  ← 栈顶（Top）
│    B    │
│    A    │
└─────────┘
          ↓
        栈底（Bottom）
```

此时栈内元素：\[A, B, C]（栈底→栈顶）

2. 再出栈 C：移除栈顶的 C，栈顶指针下移回指向 B；\*\*\*\*

```plain
┌─────────┐
│         │
│         │
│    B    │  ← 栈顶（Top）
│    A    │
└─────────┘
          ↓
        栈底（Bottom）
```

出栈后栈内元素：\[A, B]（栈底→栈顶）

注意一下栈顶指针一定是指向栈顶的元素，而不是指向栈顶再上一个的空元素

### 给定入站列为ABCD，写出至少五种可能的出栈序列

2. `A B D C`（A 出→B 出→C 入→D 入→D 出→C 出）

# 队列

#### 核心特点

1. **先进先出**先进入队列的元素，会最先离开队列。比如排队买奶茶：第一个排到的人先买完走，新来的人只能站在队尾，不能插队。
2. **两个关键操作**
   * **入队**：新元素只能从**队尾**加入队列。
   * **出队**：元素只能从**队首**离开队列。

#### 示例

##### <font style="color:rgb(0, 0, 0);">步骤 1：初始状态 - 队列为空</font>

队列里没有任何元素

```plain
[ 空 ]
```

##### <font style="color:rgb(0, 0, 0);">步骤 2：入队操作 - 甲、乙依次排队</font>

新元素只能从**队列末尾**加入

1. 甲入队\*\*\*\*

```plain
[ 甲 ]
```

2. 乙入队（站在甲后面）\*\*\*\*

```plain
[ 甲 → 乙 ]
```

##### <font style="color:rgb(0, 0, 0);">步骤 3：入队操作 - 丙、丁继续排队</font>

丙、丁依次站到队列末尾

```plain

```

此时队列顺序：甲在最前面，丁在最后面

##### <font style="color:rgb(0, 0, 0);">步骤 4：出队操作 - 按顺序办理业务</font>

元素只能从**队列最前面**离开


2. 乙接着出队\*\*\*\*

```plain
[ 丙 → 丁 ]
```

### 什么是循环结构，如何判断队列空和队列满

队列空：front == rear

队列满：(rear + 1) % maxsize == front

循环队列是对普通顺序队列的优化，解决了普通队列「假溢出」问题（数组前端有空位但无法利用）。它把存储队列的数组看成首尾相连的环形结构，通过取模运算让队头（front）、队尾（rear）指针循环移动，最大化利用数组空间。

核心特点：

* 仍遵循「先进先出（FIFO）」规则；
* 队尾指针 `rear`：指向下一个可插入元素的位置；
* 队头指针 `front`：指向当前队头元素的位置；
* 用「空出一个位置」的方式区分「队满」和「队空」（最常用方案）。

```plain
// 赫夫曼树和赫夫曼编码的存储表示
typedef struct {
    unsigned int weight;
    unsigned int parent, lchild, rchild;
}HTNode, *HuffmanTree; // 动态分配数组存储赫夫曼树
typedef char **HuffmanCode; // 动态分配数组存储赫夫曼编码表


// 求赫夫曼编码的算法如算法6.12所示
void HuffmanCoding(HuffmanTree &HT, HuffmanCode &HC, int *w, int n) {
    // w存放n个字符的权值(w>0),构造赫夫曼树HT,并求出n个字符的赫夫曼编码HC。
    if (n<=1) return;
    int m = 2 * n - 1;
    HT = (HuffmanTree)malloc((m+1) * sizeof(HTNode)); // 0号单元未用
    for (int p=1; p<=n; ++p) { // 初始化叶子结点
        HT[p].weight = w[p-1];
        HT[p].parent = 0;
        HT[p].lchild = 0;
        HT[p].rchild = 0;
    }
    for (int p=n+1; p<=m; ++p) { // 初始化非叶子结点
        HT[p].weight = 0;
        HT[p].parent = 0;
        HT[p].lchild = 0;
        HT[p].rchild = 0;
    }
    // ———— 建赫夫曼树 ————
    for (int i=n+1; i<=m; ++i) {
        // 在HT[1..i-1]选择parent为0且weight最小的两个结点,其序号分别为s1和s2。
        Select(HT, i-1, s1, s2);
        HT[s1].parent = i;
        HT[s2].parent = i;
        HT[i].lchild = s1;
        HT[i].rchild = s2;
        HT[i].weight = HT[s1].weight + HT[s2].weight;
    }
    // ———— 从叶子到根逆向求每个字符的赫夫曼编码 ————
    HC = (HuffmanCode)malloc((n+1) * sizeof(char *)); // 分配n个字符编码的头指针向量
    char *cd = (char *)malloc(n * sizeof(char)); // 分配求编码的工作空间
    cd[n-1] = '\0'; // 编码结束符
    for (int i=1; i<=n; ++i) { // 逐个字符求赫夫曼编码
        int start = n-1; // 编码结束符位置
        for (int c=i, f=HT[i].parent; f!=0; c=f, f=HT[f].parent) { // 从叶子到根逆向求编码
            if (HT[f].lchild == c) {
                cd[--start] = '0';
            } else {
                cd[--start] = '1';
            }
        }
        HC[i] = (char *)malloc((n - start) * sizeof(char)); // 为第i个字符编码分配空间
        strcpy(HC[i], &cd[start]); // 从cd复制编码(串)到HC
    }
    free(cd); // 释放工作空间
}// HuffmanCoding
```

```plain
Status TransposeSMatrix(TSMatrix M, TSMatrix &T)
{
    // 采用三元组表存储表示，求稀疏矩阵 M 的转置矩阵 T。
    T.mu = M.nu;
    T.nu = M.mu;
    T.tu = M.tu;

    if (T.tu) {
        q = 1;
        for (col = 1; col <= M.nu; ++col)
            for (p = 1; p <= M.tu; ++p)
                if (M.data[p].j == col) {
                    T.data[q].i = M.data[p].j;
                    T.data[q].j = M.data[p].i;
                    T.data[q].e = M.data[p].e;
                    ++q;
                }
    }
    return OK;
} // TransposeSMatrix
```

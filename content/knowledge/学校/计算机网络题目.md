**计算机网络复习资料**

**一．****判断题**

1.   osi模式是六层网络模型，经典协议是五层体系结构。<font style="background-color:#FFFF00;">(</font><font style="background-color:#FFFF00;">错）</font>

正确表述：OSI 模型是**七层**网络模型（物理层、数据链路层、网络层、传输层、会话层、表示层、应用层），TCP/IP 协议是**五层**体系结构（对应 OSI 五层简化），经典协议体系通常指 TCP/IP 五层。



2.   曼彻斯特编码是一种常用的物理层编码方式，它通过电压的跳变来表示辑"0"，无跳变来表示逻辑"1"。<font style="background-color:#FFFF00;">(</font><font style="background-color:#FFFF00;">错）</font>

正确规则：曼彻斯特编码通过**电压跳变**表示逻辑 “1” 和 “0”，但标准是 “位周期内前半段跳变”—— 比如 “0” 为前半高后半低，“1” 为前半低后半高（或反之，核心是**必有跳变**，无跳变不表示任何逻辑），并非 “无跳变表示 1”。



3．数据链路层中，PPP协议是点对点通信的协议。(<font style="background-color:#FFFF00;">对</font>）

PPP 协议是点对点数据链路层协议，适用于拨号、专线等场景；



4．交换机可以隔离冲突域，但不能隔离广播域。(<font style="background-color:#FFFF00;">对</font>）

交换机隔离冲突域（每个端口一个冲突域），但所有端口属于同一广播域；



5.IP地址的编址方式只能动态分配。(<font style="background-color:#FFFF00;">错</font>）

正确表述：IP 地址编址方式包括静态分配和动态分配（如 DHCP）



6.IGMP是网络层的一个协议，用于在IP主机，路由器之间传递控制消息。(<font style="background-color:#FFFF00;">对</font>）

IGMP 是网络层协议，用于组播成员管理；



7.TCP协议是面向连接的，提供可靠的数据传输服务。(<font style="background-color:#FFFF00;">对</font>）

TCP 面向连接、可靠传输（确认、重传、流量控制等）；



8.UDP协议提供拥塞控制机制。（<font style="background-color:#FFFF00;">错</font>）

错误原因：UDP（用户数据报协议）是无连接协议，仅提供 “尽力交付” 的不可靠传输服务，**不具备拥塞控制、流量控制、重传机制**，发送方会持续以自身速率发送数据，不考虑网络拥塞状态和接收方缓存能力。



9．在IP协议中，当一个段被发送后，发送方会启动一个重传计时器(TimeOut)，如果在计时器超时前没有收到确认，发送方将重新发送该段。(<font style="background-color:#FFFF00;">错</font>)

错误原因：重传计时器和确认重传机制是 TCP 协议的特性，IP 协议是无连接、不可靠的网络层协议，不提供确认和重传功能。





10．在以太网中，为了减少碰撞的可能性，发送站在发送数据前会先检查信道是否空闲，这个过程称为CSMA/CD。如果检测到信道忙，则会等待一个随机的时间后再次尝试发送。(<font style="background-color:#FFFF00;">错</font>）

正确表述：CSMA/CD 的核心是 “先听后发、边发边听、冲突停发、随机重发”，发送前检查信道空闲是 “载波监听（CS）”，但题干未提及 “边发边听” 和 “冲突检测（CD）”，且以太网的 CSMA/CD 是完整机制，题干描述不完整导致错误。



11.因特网的核心部分由大量网络和连接这些网络的路由器组成。(<font style="background-color:#FFFF00;">对</font>)

因特网核心由路由器和骨干网络组成，负责数据转发；



12.客户-服务器模式中，客户是服务的请求方，服务器是服务的提供方..(<font style="background-color:#FFFF00;">对</font>)

客户 - 服务器模式：客户请求服务，服务器提供服务；



13.卫星通信的传播时延较小。(<font style="background-color:#FFFF00;">错</font>)

错误原因：卫星通信的传播时延**很大**（而非较小）。卫星位于地球同步轨道，信号往返距离超 7 万公里，单程传播时延约 250ms，往返可达 500ms 以上，是各类通信方式中传播时延较大的一种。



14．频分复用的所有用户是在相同的时间占用不同的频率资源。（<font style="background-color:#FFFF00;">对</font>）   

频分复用（FDM）：同一时间不同频率资源分配给不同用户；

15．集线器工作在数据链路层。(<font style="background-color:#FFFF00;">错</font>)

正确结论：集线器（Hub）工作在**物理层**，仅起到信号放大和转发作用，无数据链路层的帧处理能力；交换机工作在数据链路层。



16．以太网帧中的FCS字段用于差错检测，采用CRC校验。(<font style="background-color:#FFFF00;">对</font>)



17.OSPF 属于链路状态路由算法，使用 Dijkstra 算法计算最短路径。(<font style="background-color:#FFFF00;">对</font>)

OSPF-Dijkstra算法   RIP-贝尔曼·福特算法

18.TCP 报文段 SYN 只在建立连接和释放报文中出现。(<font style="background-color:#FFFF00;">错</font>）

正确结论：TCP 报文段的 SYN 标志仅在**建立连接**时出现（第一次握手 SYN，第二次 SYN+ACK），**释放连接**时使用的是 FIN 标志，而非 SYN。





19.FTP控制链接默认使用TCP端口（<font style="background-color:#FFFF00;">错</font>）

正确表述：FTP（文件传输协议）使用两个 TCP 连接 —— 控制连接默认端口 21，数据连接默认端口 20；题干 “FIP” 为拼写错误，且未明确端口，但核心错误是 “控制链接默认使用 TCP 端口” 表述不完整且未指出正确端口，本质是对 FTP 协议连接机制的误解。



20．在TCP 拥塞控制中，阈值ssthresh一旦减小就不会再增大。(<font style="background-color:#FFFF00;">错</font>)

正确规则：TCP 拥塞控制中，ssthresh（慢启动阈值）在拥塞发生时会减小（通常减半），但在**慢启动阶段结束后进入拥塞避免阶段**，ssthresh 会保持当前值，并非 “一旦减小就不会再增大”。

**三．多选题**

1．以下属于数据链路层功能的是(<font style="background-color:#FFFF00;">AB</font>)

A．帧的封装与解封装B.差错检测(如CRC)  

C．拥塞控制        D.路由选择  

数据链路层的核心职责是处理相邻主机间的帧传输，关键功能包括帧的封装 / 解封装、差错检测与纠正、链路管理等。

+ **A. ****帧的封装与解封装**：网络层传输的 “分组” 进入数据链路层后，会添加帧头（含 MAC 地址）和帧尾（含差错检测字段）形成 “帧”；接收方收到帧后，解封装提取分组并向上层传递，这是数据链路层的基础功能。
+ **B. ****差错检测**** (****如**** CRC)**：数据在物理层传输时可能受干扰出现错误，数据链路层通过帧尾的 FCS 字段（采用 CRC 校验）检测帧传输中的比特差错，确保接收数据的完整性。
+ **C. ****拥塞控制**：用于避免网络链路因数据量过大而饱和，是传输层 TCP 协议的核心功能，与数据链路层无关。
+ **D. ****路由选择**：负责寻找跨网络的传输路径，是网络层的核心功能（由路由器实现），不属于数据链路层。



2．关于以太网(Ethernet)，下列描述正确的是(<font style="background-color:#FFFF00;">ABCD</font>)

A.使用CSMA/CD机制(在传统共享式以太网中)B.MAC地址是48位长

C.最小帧长度为64字节                       D.支持全双工通信时无需  CSMA/CD

以太网是局域网的主流技术，核心特性围绕帧结构、介质访问控制、地址格式等展开：

+ **A. ****使用**** CSMA/CD ****机制**** (****在传统共享式以太网中****)**：传统以太网（如集线器连接）中，多个主机共享传输介质，CSMA/CD（载波监听多路访问 / 冲突检测）机制通过 “先听后发、边发边听、冲突停发、随机重发” 避免帧碰撞，是共享式以太网的核心介质访问控制方法。
+ **B. MAC ****地址是**** 48 ****位长**：以太网设备的物理地址（MAC 地址）由 6 个字节（48 位）组成，前 24 位为 IEEE 分配的厂商标识（OUI），后 24 位为厂商自定义的设备标识，确保全球唯一。
+ **C. ****最小帧长度为**** 64 ****字节**：以太网帧总长度（含前导码、帧起始定界符、帧头、数据、FCS）最小为 64 字节。这一设计是为了确保发送方在帧发送完毕前，能检测到是否发生碰撞（若帧过短，发送方可能提前结束发送，无法检测碰撞）。
+ **D. ****支持全双工通信时无需**** CSMA/CD**：全双工模式（如交换机与主机连接）中，主机和交换机可同时收发数据，不存在共享介质的碰撞风险，因此无需启用 CSMA/CD 机制，传输效率更高。



3.以下协议工作在网络层的是(<font style="background-color:#FFFF00;">ABC</font>)

A.IP    B.ICMP  C.ARP   D.DNS

网络层的核心功能是跨网络路由转发、地址管理、差错控制，对应的关键协议如下：

+ **A. IP**：网际协议（IP）是网络层的核心协议，负责为数据包分配 IP 地址、定义数据包格式，并通过路由选择将数据包从源主机转发到目的主机。
+ **B. ICMP**：互联网控制消息协议（ICMP）工作在网络层，用于传递差错报告（如目标不可达、超时）、路由控制消息（如 ping 命令的请求 / 响应），辅助 IP 协议完成可靠传输。
+ **C. ARP**：地址解析协议（ARP）工作在网络层，负责将 IP 地址转换为数据链路层的 MAC 地址。因为数据链路层仅识别 MAC 地址，IP 数据包在局域网内传输时，需通过 ARP 获取目标主机的 MAC 地址才能封装成帧。
+ **D. DNS**：域名系统（DNS）负责将域名（如[www.baidu.com](https://www.baidu.com/)）转换为 IP 地址，属于应用层协议，通过 UDP 或 TCP 协议与 DNS 服务器通信，不工作在网络层。





4.关于IP地址，以下说法正确的是(<font style="background-color:#FFFF00;">ABCD</font>) 

A.A类地址的网络号占8位        B.C类地址默认子网掩码是255.255.255.0 

C.主机号全为1的地址用于广播   D.127.0.0.1是回送地址，用于本地测试

IP 地址是主机在互联网中的唯一标识，核心特性和规则如下：

+ **A. A ****类地址的网络号占**** 8 ****位**：IP 地址分为 A、B、C 三类（单播地址），A 类地址的首位为 0，网络号固定为前 8 位，主机号为后 24 位，地址范围为 1.0.0.0-126.255.255.255（127.0.0.0/8 为回送地址段）。
+ **B. C ****类地址默认子网掩码是**** 255.255.255.0**：子网掩码用于区分 IP 地址的网络号和主机号，C 类地址的网络号为前 24 位，因此默认子网掩码为 255.255.255.0（前 24 位为 1，后 8 位为 0）。
+ **C. ****主机号全为**** 1 ****的地址用于广播**：IP 地址中，主机号部分全为 1 的地址称为 “直接广播地址”，用于向该网络内的所有主机发送数据包（如 192.168.1.255 是 192.168.1.0/24 网络的广播地址）。

5.以下属于私有IP地址范围的是(<font style="background-color:#FFFF00;">ABC</font>)

私有 IP 地址是局域网内部使用的地址，不可直接访问互联网，需通过 NAT（网络地址转换）转换为公有 IP 地址才能接入互联网，标准私有 IP 地址范围如下：

+ **A. 10.0.0.0-10.255.255.255**：A 类私有地址段，子网掩码为 255.0.0.0，适用于大型局域网（如企业、校园网）。

6以太网中，MAC地址的长度不是(<font style="background-color:#FFFF00;">ACD</font>).

以太网 MAC 地址的长度是固定的 48 位（6 字节），因此 “不是” MAC 地址长度的选项分析如下：

+ **A. 32 ****位**：32 位是 IPv4 地址的长度（4 字节），用于标识互联网中的主机，与 MAC 地址（物理地址）无关。
+ **B. 48 ****位**：是以太网 MAC 地址的标准长度，因此不属于 “不是” 的范畴，排除。
+ **C. 64 ****位**：无对应主流网络地址的长度标准，既不是 MAC 地址长度，也不是 IP 地址（IPv4 为 32 位，IPv6 为 128 位）长度。

7.以下不属于OSPF协议的核心算法的是(<font style="background-color:#FFFF00;">ACD</font>).

A.距离向量算法  B。最短路径优先算法

C．生成树算法   D.CSMA/CA 算法

OSPF（开放式最短路径优先）是中大型网络常用的路由协议，其核心算法为 “链路状态算法”（又称最短路径优先算法 SPF，基于 Dijkstra 算法），各选项分析如下：

+ **A. ****距离向量算法**：是 RIP（路由信息协议）的核心算法，该算法通过交换 “距离（跳数）+ 下一跳” 的路由信息更新路由表，收敛速度慢，适用于小型网络，与 OSPF 无关。
+ **B. ****最短路径优先算法**：是 OSPF 的核心算法，OSPF 路由器通过交换链路状态信息（如链路带宽、延迟）构建全网拓扑图，再通过该算法计算到各目的网络的最短路径，因此属于 OSPF 核心算法，排除。
+ **C. ****生成树算法**：是 STP（生成树协议）的核心算法，用于解决交换机之间的环路问题，避免广播风暴，与 OSPF（路由协议）无关，属于 “不属于” 的范畴。
+ **D. CSMA/CA ****算法**：是无线局域网（如 WiFi）的介质访问控制方法，通过 “先听后发、冲突避免” 机制减少无线信号碰撞，与 OSPF 协议无关。



8.TCP协议中，有以下哪些功能(<font style="background-color:#FFFF00;">ABC</font>).

A.流量控制    B.拥塞控制    

C.可靠传输    D.不可靠传输  

TCP（传输控制协议）是面向连接的可靠传输协议，核心功能围绕 “可靠传输” 展开：

+ **A. ****流量控制**：通过 “滑动窗口” 机制实现，接收方根据自身缓存容量，在 ACK 报文中告知发送方 “接收窗口大小”，发送方调整发送速率，确保接收方不会因缓存溢出而丢失数据。
+ **B. ****拥塞控制**：通过 “拥塞窗口（cwnd）+ 慢启动阈值（ssthresh）” 机制实现，包括慢启动、拥塞避免、快速重传、快速恢复四个阶段，避免因发送速率过快导致网络拥塞。
+ **C. ****可靠传输**：通过多重机制保证数据可靠传输，包括确认机制（接收方收到数据后发送 ACK 确认）、重传机制（超时未收到确认则重传数据）、序列号（保证数据有序接收）、校验和（检测数据传输错误）。
+ **D. ****不可靠传输**：是 UDP 协议的特性，UDP 无确认、重传机制，仅提供 “尽力交付”，数据可能丢失、乱序，不属于 TCP 的功能。



9.下列协议中，属于运输层协议的是(<font style="background-color:#FFFF00;">BC</font>).

A.IP    B.TCP   

C.UDP   D.ICMP  

TCP/IP 模型的运输层负责端到端的传输服务，核心协议包括 TCP 和 UDP：

+ **A. IP**：网际协议（IP）工作在网络层，负责跨网络路由转发和 IP 地址管理，不属于运输层。
+ **B. TCP**：传输控制协议（TCP）是运输层的核心协议，面向连接、可靠传输，适用于对可靠性要求高的场景（如文件传输、网页访问）。
+ **C. UDP**：用户数据报协议（UDP）是运输层的另一个核心协议，无连接、不可靠传输，开销小、实时性强，适用于视频通话、直播等场景。
+ **D. ICMP**：互联网控制消息协议（ICMP）工作在网络层，用于传递差错报告和路由控制消息，不属于运输层。



10.IP协议组包括(<font style="background-color:#FFFF00;">ACD</font>).  

A.ARP   B.DHCP  

C.ICMP  D.IGMP

IP 协议组是围绕 IP 协议展开的一系列辅助协议，共同完成网络层的功能，核心成员如下：

·       **A. ARP**：地址解析协议（ARP），负责将 IP 地址转换为 MAC 地址，为 IP 数据包的帧封装提供支持。

·       **B. DHCP**：动态主机配置协议（DHCP），负责为网络中的主机自动分配 IP 地址、子网掩码、网关等网络配置信息，属于应用层协议，不属于 IP 协议组。

·       **C. ICMP**：互联网控制消息协议（ICMP），用于传递 IP 数据包的差错报告（如目标不可达）和路由控制消息（如 ping 请求），是 IP 协议的重要辅助协议。

·       **D. IGMP**：互联网组管理协议（IGMP），用于组播成员管理，让路由器知道局域网内哪些主机是某组播组的成员，以便转发组播数据包，属于 IP 协议组。

**3.**收发两端之间的传输距离为 1000 km，信号在媒体上的传播速率为 2×10⁸ m/s。试计算以下两种情况的发送时延和传播时延：

(1) 数据长度为 10⁷ bit，数据发送速率为 100 kbit/s。

(2) 数据长度为 10³ bit，数据发送速率为 1 Gbit/s。

从以上计算结果可得出什么结论？

核心公式

发送时延 = 数据长度（bit） / 发送速率（bit/s）

传播时延 = 传输距离（m） / 信号传播速率（m/s）

单位换算：1 kbit/s=10³ bit/s，1 Gbit/s=10⁹ bit/s，1 km=10³ m

具体解答

(1) 数据长度 = 10⁷ bit，发送速率 = 100 kbit/s

发送时延计算：

发送速率 = 100 kbit/s=100×10³ bit/s=10⁵ bit/s

发送时延 = 10⁷ bit / 10⁵ bit/s=100 s

传播时延计算：

传输距离 = 1000 km=1000×10³ m=10⁶ m

传播时延 = 10⁶ m / (2×10⁸ m/s)=0.005 s

结论：发送时延 = 100 s，传播时延 = 0.005 s

(2) 数据长度 = 10³ bit，发送速率 = 1 Gbit/s

发送时延计算：

发送速率 = 1 Gbit/s=1×10⁹ bit/s

发送时延 = 10³ bit / 10⁹ bit/s=10⁻⁶ s=1 μs

传播时延计算：

传播时延与数据长度、发送速率无关，仅由距离和传播速率决定

传播时延 = 10⁶ m / (2×10⁸ m/s)=0.005 s

结论：发送时延 = 1 μs，传播时延 = 0.005 s

结论

发送时延与数据长度正相关、与发送速率负相关；传播时延仅由传输距离和信号传播速率决定，与数据长度和发送速率无关。

低速传输、大数据量场景下，发送时延占主导；高速传输、小数据量场景下，传播时延占主导。

6.已知某主机 IP 地址为 124.12.34.56，子网掩码为 255.255.240.0。请计算：

(1) 该主机所在网络的网络地址。

(2) 该网络的广播地址。

(3) 该网络可用的主机 IP 地址范围。

(4) 该网络最多可以容纳多少台主机？

**(1) ****网络地址**

**方法**：将 IP 地址与子网掩码做**按位与运算**（二进制下，对应位均为 1 则结果为 1，否则为 0）。

IP 地址（二进制）：01111100.00001100.00100010.00111000

子网掩码（二进制）：11111111.11111111.11110000.00000000

按位与结果：01111100.00001100.00100000.00000000 → 十进制：124.12.32.0

**(2) ****广播地址**

**方法**：网络地址的**主机位（子网掩码中**** “0” ****对应的位）全置为**** 1**。

网络地址（二进制）：01111100.00001100.00100000.00000000

主机位（后 12 位）置 1：01111100.00001100.00101111.11111111 → 十进制：124.12.47.255

**(3) ****可用主机**** IP ****地址范围**

**规则**：网络地址的下一个地址～广播地址的前一个地址（排除网络地址和广播地址）。

起始地址：124.12.32.1

结束地址：124.12.47.254

**(4) ****最多容纳的主机数**

**公式**：\(2^{主机位数} - 2\)（减 2 是排除网络地址和广播地址）。

子网掩码中 “0” 的位数（主机位）：12 位（32 - 20，子网掩码前 20 位为 1）

主机数：\(2^{12} - 2 = 4094\) 台

## 实时系统（判断）
实时系统主要分为实时控制和实时信息处理，实时信息处理对于实时要求高
所有实时系统要求在物理反应时间之前就完成信息反应运算
## （共享）并行和并发的区别：
并行是在同一时刻同时发生，并发是在一个时间间隔内发生，并不一定是同时
并行是微观的，并发是宏观的
cpu和打印机同时运行：并行
## 互斥和同步
互斥和同步概念是相对的
互斥信号量初值为1
同步信号量初值为n
互斥资源和同步资源都是共享资源（判断）
## 共享资源（引申部分）
资源打印机是独占资源，不能变成共享资源
真正变成共享资源是在送到打印机之前在硬盘或者内存做一个虚拟打印机，这个虚拟打印机是可以共享的，任务送到虚拟打印机就算完成，然后逐一送到打印机打印
单独打印机是实现不了的，但是叠加硬盘或者内存就可以变成虚拟设备
硬盘和内存虚拟了打印机，链接的时候可以当成打印机用
## 虚拟和透明
虚拟是看得见但是不存在，透明是看不见但是存在。
透明是相对的，对于人类不透明可能对于其他生物是透明的。（eg：燕子是能看到磁力线的）
虚拟的核心是时分复用系统。

获取是先获取同步资源，因为同步资源多人获取没问题，但是互斥资源之后一个
两个互斥资源中间就叫邻近区
在外面同步资源多个系统可以并发，多个进程可以同时运行
生产者消费者两者权限是相同的，抢的是同一个互斥变量mutex

**必须写初始值、缩进控制好**
```
# 初始值
empty = n
full = 0
mutex = 1
in_index = 0
out_index = 0

def producer():
    while True:
        m = produce()

        P(empty)
        P(mutex)

        buffer[in_index] = m
        in_index = (in_index + 1) % n

        V(mutex)
        V(full)


def consumer():
    while True:
        P(full)
        P(mutex)

        m = buffer[out_index]
        out_index = (out_index + 1) % n

        V(mutex)
        V(empty)

        consume(m)
```

## 分段存储
分页和分段的主要区别：
- 页和块是物理单位，段是逻辑单位
- 页和块是系统定义，段是用户固定
- 分页地址是一维地址，分段地址是二维的（段名加上段内偏移量）

分页管理中，程序的逻辑地址通常写成：

```
页号 P + 页内偏移 W
```

看起来好像有两个部分，但它本质上还是一个**连续的线性地址**。
分段地址通常写成：

```
段号 S + 段内偏移 W
```

这里的段号不是系统随便拆出来的，而是程序逻辑结构的一部分。

## DMA
直接存储器访问

## 存储缓冲管理
引入缓冲的目的是
- cpu与输入输出设备间速度不匹配的矛盾
- 减少对CPU中断频率，放宽对CPU中断响应时间
- 提高CPU输入输出的运行
单缓冲，双缓冲，循环缓冲，缓冲池
缓冲池有哪几个队列：
- 空
	还未分配的缓冲
- 输入队列
- 输出队列
## SPOOLing技术（假脱机操作）
如何将独占的打印机改造成可独占的虚拟打印机
由硬盘存储空间做一个输出井，输出缓冲区，缓冲进程（井：先入后出的堆栈）
由输出进程在输出井中申请一个空闲磁盘都，将要打印的数据送入其中
输出进程再为用用户进程申请一张空白的用户申请队列。
# 6 文件管理系统
## 外存分配方式（简答）（写上优缺点）
连续分配（相当于分段）
	优点：存储空间是连续的
	缺点：容易产生外零头，需要预知文件大小
链接分配 （相当于分页）
- 隐式链接
- 显示连接
	优点：无需知道文件大小，不会产生外零头
	缺点：效率低，fat表（文件分配表）占空间
索引分配（相当于分页加上一张索引表）
- 单索引
- 多索引
	优点：直接读取，无需知道文件大小，不会产生外零头
	缺点：索引表本身占空间

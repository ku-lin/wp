# 数据库系统
数据库系统是指引入数据库后的计算机系统，一般是指由数据库、数据库管理系统（及其开发工具）、应用系统和数据库管理员组成的色存储、管理、处理和维护的系统。具有数据操纵功能。
sql的特点：
- 功能综合且风格统一
- 数据操作高度非过程化
- 面向集合的操作方法
- 以统一的语法结构提供多种使用方法

# 数据库系统的安全性
数据库的安全性是指保护数据库，以防不合法使用所造成的数据泄露、篡改或破坏。（怎么确保数据库安全性，重点关注审计和功能）
- 用户身份鉴别
- 存取控制
- 自主控制存取方法
- 授予与收回对数据的操作权限
- 数据库角色
- 强制存取控制方法
- 视图机制
- 数据加密
审计：
数据库安全审计系统提供了一种事后检查的安全机制。
审计功能把用户对数据空的所有操作自动记录下来放入审计日志。
审计功能：
- 基本功能。提供多种审计查阅方法：基础，可选，有限等。
- 提供多种审计规则。审计规则一般在数据库初始化时设定，以方便审计员管理。
- 提供审计分析和报表功能。
- 提供审计日志管理机制。主要包括为防止审计员巫山审计记录，审计日志必须先转存后删除；对转存的审计记录文件提供完整性和保密性保护；只允许审计员查阅和转储审计记录，不允许热和用户新增和修改审计记录等。

# 数据库系统的完整性
数据库的完整性是指数据库数据的正确性（完整性约束）和相容性。
完整性约束也成为完整性规则，是数据库中的数据必须满足的语义约束。

# ER图转换成逻辑结构
ER 图转换为逻辑结构，核心就是把图中的实体、属性、联系转换成关系模型，也就是数据库中的表结构。
## 一、实体转换为关系模式
ER 图中的矩形实体，通常直接转换为一张关系表。
格式为：
```
实体名(属性1, 属性2, 属性3, ...)
```
例如：
```
警员(警号, 姓名, 性别, 身份证号, 出生日期, 单位, 血型, 警衔, 入警日期, 联系电话)
```
其中能够唯一标识实体的属性作为主码，例如：
```
警号
```
## 二、属性转换为字段
ER 图中实体的属性，转换为关系模式中的字段。
例如“案件”实体有案件编号、案件来源、案件状态等属性，则转换为：
```
案件(案件编号, 案件来源, 案件状态, 案件类别, 案件副案别, 受理时间, 受理人, 受理单位, 经济损失, 受理序号)
```
其中：
```
案件编号
```
作为主码。
## 三、1 : n 联系的转换
如果两个实体之间是一对多联系，通常把 1 方的主码加入到 n 方实体中作为外码。
例如：
```
警员 1 —— 主办 —— n 案件
```
转换时，把警员的主码“警号”加入案件表中：
```
案件(..., 主办警号)
```
如果联系本身有属性，例如“主办日期”，也一起合并到 n 方实体中：
```
案件(案件编号, 案件来源, 案件状态, 案件类别, 案件副案别, 受理时间, 受理人, 受理单位, 经济损失, 受理序号, 主办警号, 主办日期)
```
其中：
```
主办警号
主办日期
```
就是由“主办”联系合并进案件表的属性。
## 四、m : n 联系的转换
如果两个实体之间是多对多联系，不能直接合并，必须单独转换成一张关系表。
例如：
```
警员 m —— 办理 —— n 案件
```
转换为：
```
办理(警号, 案件编号)
```
其中：
```
警号 + 案件编号
```
共同作为主码。
同时：
```
警号 → 警员(警号)
案件编号 → 案件(案件编号)
```
作为外码。
## 五、联系有属性时怎么处理
联系本身如果有属性，要看联系类型。
如果是 1:n 联系，联系属性通常可以合并到 n 方实体中。
例如：
```
主办日期
```
合并到案件表：
```
案件(..., 主办警号, 主办日期)
```
如果是 m:n 联系，联系属性必须放到新建的联系表中。
例如：
```
调用(部门号, 车队号, 出车日期, 车程, 费用, 车辆数目)
```
其中：
```
出车日期
车程
费用
车辆数目
```
是“调用”联系的属性。
## 六、任务一逻辑结构
根据办案管理系统 ER 图，可以转换为：
```
警员(警号, 姓名, 性别, 身份证号, 出生日期, 单位, 血型, 警衔, 入警日期, 联系电话)

案件(案件编号, 案件来源, 案件状态, 案件类别, 案件副案别, 受理时间, 受理人, 受理单位, 经济损失, 受理序号, 主办警号, 主办日期)

办理(警号, 案件编号)
```
主码：
```
警员：警号
案件：案件编号
办理：警号 + 案件编号
```
外码：
```
案件.主办警号 → 警员.警号
办理.警号 → 警员.警号
办理.案件编号 → 案件.案件编号
```
## 七、哪些属性是由联系合并来的
在任务一中：
```
主办警号
主办日期
```
是由“主办”联系合并进“案件”关系模式中的。
原因是：
```
警员 1 —— 主办 —— n 案件
```
属于一对多联系，所以把 1 方“警员”的主码加入 n 方“案件”中。
而“办理”联系是：
```
警员 m —— 办理 —— n 案件
```
属于多对多联系，不能合并，必须单独形成：
```
办理(警号, 案件编号)
```
## 八、任务四逻辑结构
车队系统可以转换为：
```
部门(部门号, 名称, 负责人)

车队(车队号, 名称, 地址)

司机(司机号, 姓名, 执照号, 电话, 工资, 车队号, 保险公司号)

车辆(车牌号, 车型, 颜色, 载重, 车队号, 保险公司号, 维修公司号)

保险公司(保险公司号, 名称, 地址)

维修公司(维修公司号, 名称, 地址)

开销(顺序号, 费用类型, 费用, 日期, 经手人, 车牌号)

调用(部门号, 车队号, 出车日期, 车程, 费用, 车辆数目)
```
主码：
```
部门：部门号
车队：车队号
司机：司机号
车辆：车牌号
保险公司：保险公司号
维修公司：维修公司号
开销：顺序号
调用：部门号 + 车队号 + 出车日期
```
外码：
```
司机.车队号 → 车队.车队号
司机.保险公司号 → 保险公司.保险公司号

车辆.车队号 → 车队.车队号
车辆.保险公司号 → 保险公司.保险公司号
车辆.维修公司号 → 维修公司.维修公司号

开销.车牌号 → 车辆.车牌号

调用.部门号 → 部门.部门号
调用.车队号 → 车队.车队号
```

# 实验2重点
## 1. 单表查询

- 基本语法：

```
select 字段名
from 表名;
```

- 指定多个字段：

```
select sno, sname
from student;
```

- 查询全部字段：

```
select *
from student;
```

## 2. 常用条件表达式

- `where`：加查询条件

```
select sno, sname
from student
where ssex = '男';
```

- 比较运算符：`=, >, <, >=, <=, <>`

```
where height > 1.8
```

- 范围查询：`between ... and ...`

```
where 入住时间 between '2019-10-01' and '2019-10-07'
```

- 模糊匹配：`like`

```
where cno like 'cc%'
```

- 集合判断：`in`

```
where sno in ('0101', '0102')
```

- 空值判断：

```
where grade is null
where grade is not null
```

- 多条件连接：

```
where height > 1.8 and ssex = '男'
```

## 3. 多表连接查询

- 当一个题目的字段分散在多张表中时，要做连接查询。
- 核心是找到表之间的关联字段。
- 例如：
    - `student.sno = sc.sno`
    - `course.cno = sc.cno`
- 旧式连接写法：

```
select student.sname, course.cno, sc.grade
from student, course, sc
where student.sno = sc.sno
  and course.cno = sc.cno;
```

- 考试重点：
    - 先看题目要哪些字段
    - 再看这些字段在哪些表里
    - 最后补上连接条件

## 4. 子查询

- 子查询就是把一个查询结果作为另一个查询的条件。
- 常见位置：`where` 后面。
- 例子：

```
select sname
from student
where sno in (
    select sno
    from sc
    where cno like 'dd%'
);
```

- 适合“先找一批编号，再查详细信息”的题。
- 考试常考：
    - `in (select ...)`
    - 先筛课程，再筛学生
    - 先筛住宿记录，再查人员信息

## 5. 分组与聚合统计

- 分组：`group by`
- 聚合函数：
    - `count()` 计数
    - `avg()` 平均值
    - `max()` 最大值
    - `min()` 最小值
- 例子：

```
select sno, count(cno), avg(grade)
from sc
where grade is not null
group by sno;
```

- 含义：按 `sno` 分组，统计每个学生的选课门数和平均成绩。
- 常见题型：
    - 每个学生……
    - 每门课程……
    - 每个号码……
    - 每个身份证号……

## 6. 分组后过滤

- `where` 是分组前过滤。
- `having` 是分组后过滤。
- 例子：

```
select sno
from sc
group by sno
having count(*) >= 3;
```

- 含义：筛选出分组后记录数不少于 3 的学生。
- 常见搭配：

```
having min(grade) >= 80
having count(*) = 4
having count(cno) >= 1
```

- 考试很爱考 `where` 和 `having` 区别。

## 7. 时间范围查询

- 两种常见写法：

```
where 入住时间 between '2019-10-01' and '2019-10-07'
```

```
where 入住时间 >= '2019-12-01'
  and 入住时间 < '2020-01-01'
```

- 第二种写法更适合查整月数据，边界更清晰。
- 考试要注意：
    - 日期字段能不能直接比较
    - 起止范围是否包含端点
    - 是否跨月、跨天

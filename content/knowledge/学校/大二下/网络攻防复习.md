# 判断
1. 在DVWA等靶场中，安全等级越高通常意味着服务端过滤和校验逻辑越严格.(对)

2. ARP协议缺少身份认证机制，因此攻击者可能伪造ARP响应影响局域网主机的IP-MAC映射。(对)

3. TCP SYN扫描在发现开放端口后通常会完成完整三次握手，因此与TCP Connect扫描完全相同。(错)
connect是全扫描，SYN是半开扫描，只发SYN包，之后直接拒绝
4. FTP的控制连接和数据连接一定使用同一个TCP端口。（错）
数据端口是21
控制端口是22
5. Wireshark若要解密多数 TLS流量，通常需要导入会话密钥日志等额外材料 （对）
实验2，https解密需要加密
6. 口令暴力破解之一字典大小有关，域账户锁定，验证码，速率限制等防护无关 （错）
账户被锁定后密码爆破是打不开的
7. 反向连接木马由被控端主动连接控制端，在NAT或防火墙场景中更常见。 （对）
实验四，因为内外网交互的难度不一样
8. 栈Canary的主要作用是在函数返回前检测栈缓冲区溢出是否破坏了保护 （对）
canary是在返回地址前面加入一个值来判断值是否被篡改，通过判断这个值来确定有咩有被篡改
9. 使用参数化查询可以有效降低SQL注入风险。（对）
参数化查询能避免拼接sql语句
10. 存储型XSS的攻击代码只存在于URL参数中，不会保存在服务器端数据中 （错）
存储型是要保存的，反射形值保存在url中
11. 虚拟化靶场的快照功能会自动阻止漏洞利用，因此无需恢复或加固环境 （错）
快照只能用于恢复环境，和漏洞利用无关
12. ICMP仅能用于ping测试，不能辅助网络诊断或攻击流量分析。（错）
ICMP还能应用于枚举到目的地的每个点
13. Hydra可通过指定用户名列表和密码列表对授权目标服务进行口令强度测式 （对）
Hydra用于破解windows远程桌面，也可以用来测试强度
14. Cobalt Strike Beacon的通信方式只能采用DNS，不能通过HTTP/HTTPS回连 （错）
实验中是通过https连接
15. 堆区、栈区、代码段和BSS段都属于进程虚拟地址空间的一部分。 （对）
堆溢出实验
16. 格式化字符串漏洞只会造成程序崩溃，不可能导致信息泄露。(错)
通过%h能做到任意地址读取，如果加上%n就能对于任意地址进行写
17. ASLR通过随机化地址布局提高攻击者预测关键地址的难度。(对)
ROP实验，地址空间布局随机化（ASLR）
18. SQL报错注入依赖页面或接口能够向攻击者暴露可利用的数据库错误信息（对）
需要有地方把数据库的村务信息打印出来，不然看不到报错信息
19. DOM型XSS 的触发点一定在服务器端模板中，与浏览器端JavaScript 代码无关(错)
DOM指HTML中的框架，可以控制HTML中的节点，需要使用js代码
整个XSS都需要用js代码
20. 在生产系统中，上传文件保存目录与代码执行目录放在一起有助于降低风险.（错）
这个目录会被赋予执行权限，造成上传文件可执行

四、简答题

1. 说明Nmap主机发现、端口扫描、服务版本探测在目标信息收集中的作用。
主机发现的作用是确认资产的存活
端口扫描是用来识别开放服务的入口
服务版本探测的作用是在扫描服务的基础上对服务的版本进行确定
加在一起是为了后续的漏洞验证和风险评估验证提供依据

2. 简述ARP欺骗造成中间人攻击的基本过程，并列举两项防御措施。
arp是伪造ARP，然后目标机器和网关产生错误的ip和mac映射，攻击者可以通过中间人进行转发。
静态绑定arp和ip的mac。
在交换机上用dai，动态监测arp包的真伪，通过DHCP Snooping 绑定表匹配ip和mac地址

3. 说明文件上传漏洞中“前端校验、MIME校验、文件头校验、白名单”的安全强度差异。
前端校验最弱，可以直接抓包绕过，也可以更改前端代码。
MIME我们可以伪造content-tag来进行绕过
文件头校验可以通过伪造文件头，在上传包中加一个图片然后放木马绕过
白名单防护能力最强，只有文件后缀名符合白名单的才能绕过

4. 说明Wireshark分析HTTPS流量时导入Pre-master secret日志的作用
这个是记录浏览器与服务器的浏览密钥，wireshark在导入密钥后可以解密TLS密钥，然后我们就可以导出http明文对象

5. 简述Hydra远程口令测试的关键参数类型，并说明应如何避免对业务造成影响。
`-l zuziyi`指定单个用户名为 `zuziyi`
`-P misc2.txt`指定密码字典文件，从 `misc2.txt` 中逐个读取密码尝试
`rdp://`指定测试的服务类型为 RDP 远程桌面服务
`192.168.70.144`指定目标主机 IP 地址
`-V`显示详细测试过程，输出每一次尝试的用户名和密码组合
控制并发量，控制速率，以防对服务器进行损害，注意访问响应

6. 简述格式化字符串漏洞从信息泄露到任意写的基本思路。
格式化字符串漏洞的基本原因是程序把用户输入直接作为 `printf` 等函数的格式化字符串使用，例如 `printf(user_input)`，导致用户输入中的 `%x`、`%s`、`%n` 等格式符被程序解析执行。
利用时通常先通过 `%x`、`%p` 等格式符读取栈上的数据，泄露栈地址、程序地址或 libc 地址，并通过输入特征值如 `AAAA` 判断用户输入在格式化参数中的偏移位置。获得偏移后，攻击者可以把目标地址放入输入内容中，再利用 `%n` 将“当前已经输出的字符数”写入该地址。通过控制输出字符数量，就可以控制写入的数值。
五、编程解读题

1. 阅读程序回答问题

```c
#include<stdio.h>
#include<string.h>
void log_name(char *name) {
	char buf[64];
	strcpy(buf, name);
	printf("hello %s\n", buf);
}
int main(int argc, char *argv[]) {
	if (argc < 2) {
		printf("usage: %s <name>\n", argv[0]);
		return 0;
	}
	log_name(argv[1]);
	return 0;
}
```

（1）指出该程序存在的主要安全问题。（5分）
栈溢出漏洞。
（2）给出造成问题的核心代码或逻辑，并解释漏洞形成原因。（5分）
argv[1]是用户控制，同时存在strcpy存在复制字符串，如果name大于64字符就能触发栈溢出控制。
（3）结合课程实验说明可能的利用思路或安全加固方案。（5分）
利用思路：通过GDB找到输入点到注入点的偏移量，然后去覆盖这个返回地址，如果没有开NX，那么直接asm创建一段shellcode，如果开了NX，就只能找到后门函数或者是通过泄露puts等函数的地址算出libc基址，最后调用shellcode然后输出。
安全加固方案最短就是加上canary，或者不使用strcpy这种无边界检查的函数，使用strncpy（）等带有边界检查的函数。

漏洞形成原因主要有三点：  
第一，`$_FILES["file"]["type"]` 来自客户端请求，攻击者可以伪造，比如把 PHP 脚本文件的类型伪造成 `image/jpeg`。  
第二，程序直接使用原始文件名 `$name` 保存文件，没有限制后缀名，也没有重命名，攻击者可能上传 `.php`、`.phtml` 等可执行脚本文件。  
第三，上传目录如果位于 Web 可访问目录下，并且服务器会解析其中的脚本文件，攻击者访问上传后的文件时就可能执行恶意代码。
（3）结合课程实验说明可能的利用思路或安全加固方案。
结合课程实验，可能的利用思路是：攻击者构造一个带有脚本代码的文件，例如伪装成图片类型的脚本文件，并在上传请求中修改 `Content-Type` 为 `image/jpeg` 或 `image/png`，从而绕过程序的简单判断。如果上传成功并且服务器解析该文件，攻击者访问上传路径时可能执行脚本代码，进一步获取服务器信息或控制权限。

传统的不能成功，因为开了NX，不能在栈上面直接执行函数。
能成功，因为开NX，但是并不需要直接再栈上面执行函数，这里是直接传入地址执行，直接传入system函数和"/bin/sh"跟在栈的返回地址后面就行。没有开启Canary说明可以直接栈溢出，没有开PIE说明system函数的地址不会随机化加载。

3. 某靶场Web服务器开启80端口后，短时间内无法正常访问。抓包和主机状态显示：目标持续收到大量 TCP SYN 报文，源 IP 随机变化，服务器存在大量 SYN\_RECV 半连接，CPU 不是很高但新连接建立困难。请综合分析该现象对应的拒绝服务攻击类型及其原理（3分），说明如何用 Hping3 构造实验流量（3分），并用系统状态或抓包结果验证效果（3分），比较其与 UDP Flood、Smurf 攻击、Land 攻击的区别（3分），并提出针对性的防护与缓解措施（3分）。
该现象对应的攻击类型是 TCP SYN Flood 拒绝服务攻击。其原理是攻击者向目标 80 端口持续发送大量 TCP SYN 报文，服务器收到 SYN 后会为连接分配半连接资源，并回复 SYN+ACK，等待客户端最后的 ACK。如果攻击者伪造随机源 IP 或不完成三次握手，服务器会长时间保留大量 SYN_RECV 半连接，导致半连接队列被占满，正常用户的新连接难以建立。题目中“源 IP 随机变化”“大量 SYN_RECV 半连接”“CPU 不高但新连接困难”正是 SYN Flood 的典型特征。
```
ping3 -S -p 80 --rand-source -i u1000 192.168.70.144
```

其中，`-S` 表示设置 TCP SYN 标志位，`-p 80` 表示目标端口为 80，`--rand-source` 表示随机伪造源 IP，`-i u1000` 表示每隔 1000 微秒发送一个包。实验中也可以使用 `-c` 指定发送数量，避免流量过大影响环境。
`ss -ant | grep SYN-RECV`可以观察系统中的包。在 Wireshark 中可以看到目标持续收到大量 SYN 报文，源 IP 不断变化，但很多连接没有完成三次握手中的最后 ACK。此时正常访问 Web 页面会变慢或无法建立新连接。还可以通过wireshark的统计功能查看包的数量
SYN Flood 主要利用 TCP 三次握手机制，占用服务器半连接队列资源；UDP Flood 是大量发送 UDP 数据包，主要消耗带宽、主机处理能力或触发 ICMP 不可达响应；Smurf 攻击利用 ICMP 广播放大，攻击者伪造受害者 IP 向广播地址发送 ICMP 请求，使大量主机向受害者回复；Land 攻击则构造源 IP 和目标 IP 相同、源端口和目标端口相同的异常数据包，可能导致目标系统协议栈异常或资源消耗。
开启 SYN Cookie，使服务器在半连接队列压力较大时不立即分配完整连接资源，通过防火墙，通过设备过滤伪造的ip，使用公网运营商的流量清洗。

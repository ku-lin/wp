## vmcode
`qword_40E8 = (__int64)malloc(0x1000u);`
创建了一个堆
`v4 = read(0, (void *)qword_40E8, 0x1000u);`
直接把你输入的东西放进堆里
然后写24字节的空，直接寄存器清零
之后就是解析输入的opcode，生成机械吗
最后开一个seccomp，只开了几个syscall，但是还是可以直接打开根目录下的flag文件
开了open,sendfile等，但是这两个其实就够用
之后就是指令集的问题了，找一下指令集
```
op 0: load_imm reg, imm64
op 1: add dst, src
op 2: get_pc reg
op 3: store_byte offset_reg, value_reg
op 4: jump reg
op 5: ret
```
op2能直接给你栈偏移
op3能直接把寄存器写到指定的偏移
op4能跳转
那么利用思路就清晰了
op2拿到偏移
op0准备一个常量
op0再找一个bss写shellcode
op1把指向shellcode写入的位置
把shellcode循环op0，op3，op1给他放进去
最后直接op4跳转就能执行

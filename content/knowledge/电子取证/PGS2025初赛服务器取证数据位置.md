## 挂载和仿真要先单独处理

这套服务器检材不是普通 Linux 盘直接打开就能做完，核心顺序是：

1. 先挂载 `server1.001`，确认它是 Unraid OS 启动盘。
2. 再挂载或解密 `server2.001`，它是 LUKS 加密的数据盘，解密后是 ZFS。
3. 数据盘解密后，再挂载里面的 `docker.img`，Docker 的容器、镜像、日志、卷数据都在这里。
4. 如果选择仿真，需要把 Unraid 启动盘按 USB 设备处理，否则可能无法正常从 Unraid 启动。
5. Windows 服务器部分不在这两个服务器镜像里，而是在贾韦码 Windows 计算机检材内的虚拟机中。

Unraid 官方说明里强调：Unraid 需要写入 USB flash device，并从 USB 设备启动；USB 设备还需要具备唯一 GUID。因此仿真时不能只把启动盘当普通 IDE/SATA 硬盘看，启动设备类型也会影响系统识别。

### Docker 虚拟硬盘挂载

数据盘解密后会看到 `docker.img`。它不是普通文件附件，而是 Docker 的虚拟硬盘，需要再次作为磁盘镜像挂载。

挂载后重点路径：

```text
/image/btrfs/imagedb/content/sha256
/containers
/volumes
```

常见证据：

- 镜像元数据：`/image/btrfs/imagedb/content/sha256`
- 容器配置：`/containers/<container_id>/config.v2.json`
- 端口映射：`/containers/<container_id>/hostconfig.json`
- 容器日志：`/containers/<container_id>/<container_id>-json.log`
- MySQL 数据卷：`/volumes/www_mysql_data/_data`

### VMware 仿真时的 USB 盘处理

题解引用了 VMware 虚拟 USB 盘的做法：编辑虚拟机的 `.vmx`，让某个 VMDK 以 USB disk 形式挂载。思路是先确保 EHCI 控制器存在，再加入 `ehci:0` 设备配置。

参考配置：

```text
ehci.present = "TRUE"
ehci.pciSlotNumber = "35"

ehci:0.present = "TRUE"
ehci:0.deviceType = "disk"
ehci:0.fileName = "disk0-000002.vmdk"
ehci:0.readonly = "FALSE"
```

如果原来把启动盘挂在 IDE 上，需要移除或禁用类似配置：

```text
ide0:0.filename = "disk0-000002.vmdk"
ide0:0.present = "TRUE"
```

启动时若没有自动引导，可进 BIOS/启动菜单选择 `EFI USB Device`。

### 仿真后手动处理网络和磁盘

仿真成功后，如果 WebUI 不能直接访问，先看网络：

```bash
ip addr
ip route
dhcpcd
```

也可以临时给 `br0` 加 IP：

```bash
ip addr add <VMware NAT IP> dev br0
```

如果 Unraid 保存了旧磁盘识别符，数据盘可能无法直接上线。处理顺序：

1. 在 WebUI 清理旧磁盘配置。
2. 重新分配数据盘。
3. 上传或指定 keyfile。
4. 启动阵列。
5. 再启动 Docker 容器。

## 命令速查：挂载、Docker、SQL

下面命令按 Linux 取证环境写，路径按实际情况替换。Windows 上已经用 Arsenal Image Mounter、FTK Imager、取证大师、UFS Explorer 挂出来的盘，也可以直接跳过前面的镜像挂载步骤，从对应盘符读取文件。

### 查看镜像分区

先确认镜像里有哪些分区、起始扇区、文件系统：

```bash
fdisk -l server1.001
fdisk -l server2.001
parted server1.001 unit s print
parted server2.001 unit s print
mmls server1.001
mmls server2.001
```

如果已经转换成 VMDK/VHD，也可以用取证工具看分区；Linux 下更推荐用 `qemu-nbd` 或 `guestmount`。

### 挂载 Unraid 启动盘

方式一：用 `losetup -P` 自动识别分区：

```bash
sudo losetup --find --partscan --show server1.001
lsblk
sudo mkdir -p /mnt/unraid
sudo mount -o ro /dev/loop0p3 /mnt/unraid
```

如果分区 3 挂不上，先用 `lsblk -f` 确认分区号和文件系统：

```bash
lsblk -f /dev/loop0
sudo mount -o ro /dev/loop0p1 /mnt/unraid
sudo mount -o ro /dev/loop0p2 /mnt/unraid
sudo mount -o ro /dev/loop0p3 /mnt/unraid
```

方式二：按 offset 手动挂载。假设 `fdisk` 看到分区起始扇区是 `<START_SECTOR>`，扇区大小是 512：

```bash
sudo mkdir -p /mnt/unraid
sudo mount -o ro,loop,offset=$((<START_SECTOR>*512)) server1.001 /mnt/unraid
```

卸载：

```bash
sudo umount /mnt/unraid
sudo losetup -d /dev/loop0
```

### 挂载 VMDK/VHD

VMDK/VHD 推荐用 `qemu-nbd`：

```bash
sudo modprobe nbd max_part=16
sudo qemu-nbd --read-only --connect=/dev/nbd0 1.vmdk
lsblk -f /dev/nbd0
sudo mkdir -p /mnt/unraid
sudo mount -o ro /dev/nbd0p3 /mnt/unraid
```

卸载：

```bash
sudo umount /mnt/unraid
sudo qemu-nbd --disconnect /dev/nbd0
```

如果系统装了 `libguestfs`，也可以：

```bash
guestfish --ro -a 1.vmdk -i
guestmount --ro -a 1.vmdk -i /mnt/unraid
fusermount -u /mnt/unraid
```

### 解密 LUKS 数据盘

先挂载数据盘镜像并识别分区：

```bash
sudo losetup --find --partscan --show server2.001
lsblk -f /dev/loop1
sudo cryptsetup luksDump /dev/loop1p1
```

使用口令或 keyfile 解密：

```bash
sudo cryptsetup luksOpen /dev/loop1p1 unraid_data
```

如果有密钥文件：

```bash
sudo cryptsetup luksOpen /dev/loop1p1 unraid_data --key-file ./keyfile
```

如果拿到的是 Base64 字符串，先写成临时文件再试：

```bash
printf '%s' '<BASE64_OR_PASSWORD>' > keyfile.txt
sudo cryptsetup luksOpen /dev/loop1p1 unraid_data --key-file ./keyfile.txt
```

注意：有些工具会把 Base64 字符串当“密码”输入，而不是先 Base64 解码；两种方式都可以试，但要记录清楚你使用的是原字符串还是解码后的字节。

### 挂载 ZFS 数据池

LUKS 打开后查看映射设备：

```bash
lsblk -f /dev/mapper/unraid_data
```

查看 ZFS 池：

```bash
sudo zpool import
```

只读导入，避免改动检材：

```bash
sudo mkdir -p /mnt/zfs
sudo zpool import -o readonly=on -o altroot=/mnt/zfs <pool_name>
zfs list
```

如果提示曾在其它系统使用，可加 `-f`：

```bash
sudo zpool import -f -o readonly=on -o altroot=/mnt/zfs <pool_name>
```

导出和关闭：

```bash
sudo zpool export <pool_name>
sudo cryptsetup luksClose unraid_data
sudo losetup -d /dev/loop1
```

### 挂载 docker.img

先确认文件类型：

```bash
file /mnt/zfs/docker.img
sudo losetup --find --partscan --show /mnt/zfs/docker.img
lsblk -f /dev/loop2
```

如果 `docker.img` 本身就是 BTRFS：

```bash
sudo mkdir -p /mnt/docker
sudo mount -o ro,loop /mnt/zfs/docker.img /mnt/docker
```

如果 `losetup -P` 识别出了分区：

```bash
sudo mkdir -p /mnt/docker
sudo mount -o ro /dev/loop2p1 /mnt/docker
```

BTRFS 有时需要加恢复/降级选项：

```bash
sudo mount -t btrfs -o ro,loop,recovery /mnt/zfs/docker.img /mnt/docker
sudo mount -t btrfs -o ro,degraded /dev/loop2p1 /mnt/docker
```

卸载：

```bash
sudo umount /mnt/docker
sudo losetup -d /dev/loop2
```

在 Unraid 仿真系统里，先确认阵列和数据盘已经挂载，再启动 Docker：

```bash
df -h
mount | grep -E 'disk1|docker|zfs'
/etc/rc.d/rc.docker start
docker ps -a
```

如果 Docker 已经启动但容器未启动：

```bash

docker start $(docker ps -aq)
```

进入容器：

停止 Docker：

### SQL 查询模板

查看库、表、字段：

```sql
SHOW DATABASES;
USE pgs;
SHOW TABLES;
SHOW FULL COLUMNS FROM info;
SHOW FULL COLUMNS FROM user;
```

模糊找表名：

```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = DATABASE()
  AND (
    table_name LIKE '%user%'
    OR table_name LIKE '%member%'
    OR table_name LIKE '%bank%'
    OR table_name LIKE '%card%'
    OR table_name LIKE '%recharge%'
    OR table_name LIKE '%withdraw%'
    OR table_name LIKE '%order%'
    OR table_name LIKE '%pay%'
  );
```

模糊找字段名：

```sql
SELECT table_name, column_name, column_comment
FROM information_schema.columns
WHERE table_schema = DATABASE()
  AND (
    column_name LIKE '%status%'
    OR column_name LIKE '%clock%'
    OR column_name LIKE '%money%'
    OR column_name LIKE '%amount%'
    OR column_name LIKE '%card%'
    OR column_name LIKE '%bank%'
    OR column_name LIKE '%alipay%'
  )
ORDER BY table_name, ordinal_position;
```

查看表样例：

```sql
SELECT * FROM info LIMIT 20;
SELECT * FROM user LIMIT 20;
SELECT * FROM user_member LIMIT 20;
```

金额统计模板：

```sql
SELECT SUM(money) FROM <table_name> WHERE status = <status_value>;
SELECT SUM(amount) FROM <table_name> WHERE status = <status_value>;
```

按姓名、银行卡查：

```sql
SELECT * FROM user WHERE name = '<姓名>';
SELECT * FROM <bank_table> WHERE name = '<姓名>' OR username = '<用户名>';
```

## 总排查清单

### 第一步：确认盘和挂载层级

```text
server1.001 -> Unraid 启动盘 -> /config、/logs
server2.001 -> LUKS + ZFS 数据盘 -> /docker.img、/www
docker.img  -> Docker BTRFS 虚拟硬盘 -> /containers、/image、/volumes
Windows VM  -> 账本系统 -> D:\crm、MongoDB
```

### 第二步：查基础配置

```bash
cat /mnt/unraid/config/go
cat /mnt/unraid/config/network.cfg
cat /mnt/unraid/config/disk.cfg
cat /mnt/unraid/config/docker.cfg
cat /mnt/unraid/config/shadow
```

### 第三步：查诊断包

```bash
unzip tower-diagnostics-*.zip -d tower-diagnostics
grep -RIn "GUID\|UUID\|version\|zfs\|luks\|docker\|rc.docker\|br0\|gateway" tower-diagnostics
```

### 第四步：查 Docker

```bash
find /mnt/docker/containers -maxdepth 2 -type f | sort
grep -RIn "\"Name\"\|\"Image\"\|PortBindings\|MYSQL\|Apache\|login" /mnt/docker/containers
find /mnt/docker/image/btrfs/imagedb/content/sha256 -type f | sort
```

### 第五步：查网站和数据库

```bash
grep -RIn "DB_HOST\|DB_NAME\|DB_USER\|DB_PWD\|password\|login\|admin" /mnt/zfs/www
find /mnt/zfs/www/mysql_data -type f -name "*.sql" -maxdepth 1
```

### 第六步：查 Windows 虚拟机项目

```powershell
Get-ChildItem -LiteralPath 'D:\crm' -Recurse -File |
  Select-String -Pattern 'MONGODB_URI|createCipheriv|AES_SECRET_KEY|JWT_SECRET|MASTER_PASSWORD|Customers|base64|utf8'
```

## 易漏点

- Unraid 启动盘要按 USB 盘思路处理，GUID 和启动标识不是普通 Linux 根分区里的 `/etc` 信息。
- 数据盘要先过 LUKS，再看 ZFS；没解密时看不到 `/www` 和 `docker.img`。
- `docker.img` 还要二次挂载，容器配置和日志不在启动盘 `/config` 里。
- 容器名、端口、挂载点优先看 `config.v2.json` 和 `hostconfig.json`。
- 网站后台入口不一定在源码里显眼，应用日志和 Web 容器 json 日志更好找。
- SQL 备份和生成脚本可能比数据库当前数据更完整，尤其是提现、充值、批量生成记录。
- Windows 服务器在另一个检材的虚拟机里，别在 Unraid 两块盘里硬找 `D:\crm`。
- JS 混淆题先去混淆，再看真实控制流、编码、密钥优先级。
